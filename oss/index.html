If you arrived from a pull request

A human reviewed that PR before it reached your repository.

Fabrik is an operator-reviewed autonomous engineering system. Agents draft the code. An operator reads every diff, runs the tests, and decides whether to submit. Nothing is opened upstream on autopilot.

Before submission

  1. 01Agent drafts the change
  2. 02Tests and acceptance gates run
  3. 03Operator reviews the diff
  4. 04Operator submits or discards

You — the maintainer — review again on your side. Merge is yours.

01 — What this is

Composable agents, one human at the gate.

Fabrik runs a small workforce of coding agents against tickets. The output is always a pull request, never a direct commit. An operator sits between the agent and your repository and is accountable for what gets submitted.

We are not a swarm. We are not a bot account scraping issues. We submit slowly, narrowly, and at most one PR per repository — and we are not looking to flood maintainers.

02 — Don't want PRs from Fabrik?

Tell us to stop. We stop.

Pick whichever is easiest. Either is enough.

  • Reply on the pull request

    Ask us to close it and add your repository to our skip-list. The operator reads every comment.

  • Add a line to CONTRIBUTING.md

    The phrase no-ai-prs anywhere in CONTRIBUTING.md, README.md, or repository topics removes you from discovery permanently.

03 — Contact

Talk to the operator, not a form.

The fastest path is a comment on the pull request itself — the operator reads every reply. If the PR is already closed, opt out via the CONTRIBUTING.md path above and we won't be back.
